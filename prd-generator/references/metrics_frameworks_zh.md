# 产品指标框架速查（中文）

撰写 **1.4 成功指标** 或 **七、验收标准** 时选用合适框架，指标描述须用**简体中文**并给出可量化目标。

---

## 框架选择

| 场景 | 推荐框架 |
| --- | --- |
| 增长型 C 端产品 | AARRR |
| 体验与效率型功能 | HEART |
| 单一核心价值产品 | 北极星指标 |
| 战略级项目对齐 OKR | OKR |
| B 端/平台/集成类 | 自定义 KPI + 可靠性指标 |

---

## AARRR（海盗指标）

| 阶段 | 中文 | 示例指标 |
| --- | --- | --- |
| Acquisition | 获客 | 注册转化率、渠道 UV |
| Activation | 激活 | 首日完成核心动作比例 |
| Retention | 留存 | 7 日/30 日留存 |
| Revenue | 收入 | ARPU、付费转化率 |
| Referral | 推荐 | 邀请成功率、K 因子 |

---

## HEART

| 维度 | 中文 | 示例 |
| --- | --- | --- |
| Happiness | 满意度 | NPS、CSAT |
| Engagement | 参与度 | 周活跃、功能使用频次 |
| Adoption | 采纳 | 新功能 30 日内使用率 |
| Retention | 留存 | 任务重复完成率 |
| Task Success | 任务成功 | 流程完成率、错误率 |

---

## 北极星指标

- 选一个最能代表用户获得核心价值的指标
- 示例：平台能力 →「业务系统成功完成核心操作的比例」
- 示例：数据同步 →「目标系统可查询到完整权威数据的比例」

---

## OKR 写法

**目标（O）：** 定性、鼓舞人心  
**关键结果（KR）：** 可量化、有时限

示例：

- O：建立稳定可信的业务数据底座  
- KR1：第一期上线后 30 天内核心链路成功率 ≥ 99.5%  
- KR2：核心查询 P95 延迟 < 500ms  

---

## B 端 / 集成类常用指标

| 类别 | 指标示例 |
| --- | --- |
| 可靠性 | 同步成功率、接口可用性、幂等重试成功率 |
| 效率 | 单次同步耗时、运维处理工单数 |
| 质量 | 数据一致性对账差异率、字段完整率 |
| 采纳 | 接入系统数、API 调用量 |

详细英文原文见 `references/metrics_frameworks.md`。
